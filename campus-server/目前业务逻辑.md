# campus-server 最终版修正说明

## 当前登录逻辑

- `POST /auth/send-code`
  - 校验校园邮箱
  - 生成验证码
  - 写入 `email_verification_codes`
  - 发送邮件

- `POST /auth/verify`
  - 校验最新一条未使用验证码
  - 第一次登录时创建 `users` 记录
  - 已存在用户则更新 `last_login_at`
  - 返回：
    - `token`
    - `user`
    - `needProfileCompletion`
    - `nextPage`
    - `isFirstLogin`

- `PUT /users/me`
  - 更新昵称、年级、院系、头像、身份模式
  - 自动计算 `profileCompleted`
  - 返回 `nextPage`

## 自动初始化

服务启动时会自动：

- 创建数据库 `campus_app`（如果不存在）
- 创建 `users` 表（如果不存在）
- 创建 `email_verification_codes` 表（如果不存在）

## 运行前仍需保证

- MySQL 已启动
- `.env` 中数据库账号密码正确
- `.env` 中 SMTP 正确
- 当前 MySQL 用户拥有建库建表权限
